## Development Tools & Visualizers

### sim-core Web Visualizer

A lightweight browser-based viewer for sim-core's physics simulation. Streams JSON snapshots via WebSocket and renders entities on a canvas. Great for debugging field layouts and piece behavior without AdvantageScope.

**Location:** `robot/sim-core/examples/web-visualizer/` (scaffold)

**Quick start:**
```bash
cd robot/sim-core/examples/web-visualizer
node server.js
# Open http://localhost:8080 in a browser
```

See the **Simulation System Design → Visualization & Web UI** section for implementation details and snapshot format.

### Telemetry Tools

- **AdvantageScope** — primary 3D visualization and log replay tool (external)
- **Shuffleboard** — WPILib dashboard for real-time subsystem status, motor temps, test results
- **Diagnostic Server** — RTU's embedded web server (port 5800) for Chart.js test result charts

### Build & Deploy Scripts

| Script | Location | Purpose |
|---|---|---|
| `gradlew build` | `robot/<year>/` | Compile and run unit tests |
| `gradlew deploy` | `robot/<year>/` | Deploy to roboRIO via USB or WiFi |
| `gradlew simulateJava` | `robot/<year>/` | Run physics simulation locally |
| `npm run dev` | `operations-manual/` | Dev server for documentation site |
| `npm run build` | `operations-manual/` | Production build of documentation |
| `npx http-server` | `g201/calculator/` | Serve the G201 calculator locally |

## Programming Guides & Patterns

### Short Recipes (in FRC Cookbook)

The **FRC → Programming Cookbook** section has actionable, copy-paste recipes for common programming tasks:

- Defining game pieces (sphere, cylinder, box configs)
- Setting up intake zones and launching
- Building simulation for a new game year
- Writing @RobotAction automated tests
- Creating new subsystems from the template

### Architecture Patterns We Use

| Pattern | Where It's Used | Why |
|---|---|---|
| **Command-based** | All robot subsystems | WPILib standard — separates "what to do" from "when to do it" |
| **Builder pattern** | `GamePieceConfig`, `ChassisConfig` | Clean construction of objects with many optional parameters |
| **Counter-based sim** | Intake/launch system | Avoids simulating pieces inside the robot — just count them |
| **Proximity sleep/wake** | `GamePieceManager` | Only simulate pieces near the robot for performance |
| **Annotation-driven testing** | `@RobotAction` | Auto-discovers test methods without manual registration |
| **Dual-engine simulation** | MapleSim + ODE4J | Specialized engines for drivetrain vs. game piece physics |

### Version Control Conventions

- **Branch naming**: `feature/<description>`, `fix/<description>`, `comp/<event-name>`
- **Commit messages**: imperative tense — "Add intake subsystem", not "Added intake subsystem"
- **Pre-competition**: create a `comp/` branch, deploy from it, never push untested code to `main` during an event
