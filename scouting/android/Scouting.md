# Scouting Manual: QR Code Data Output

This manual documents the format of the data generated by the Scouting App's QR Code.

The data encoded within the QR Code is a single CSV string generated by concatenating `MatchRecord`, `AutonPoints`, `TeleOpPoints`, and `EndPoints`. Before being displayed, this string is compressed into a Base64 format (as seen in `QrGenerator.dart`).

Below is the complete sequence of variables, exactly as they are ordered in the CSV output.

### 1. General Match Information (`MatchRecord`)
1. **`Team`** (String) - The team number being scouted.
2. **`MatchKey`** (String) - The Blue Alliance match key (e.g., `2026mibg_qm1`).
3. **`MatchNumber`** (Integer) - The match number.
4. **`ScouterName`** (String) - The name of the scouter.
5. **`AllianceColor`** (String) - The color of the alliance (`Red` or `Blue`).
6. **`EventKey`** (String) - The event key.
7. **`Station`** (Integer) - Driver station number (e.g., 1, 2, or 3).
8. **`BatteryPercentage`** (Integer) - The device's battery percentage at the time of recording.

### 2. Autonomous Data (`AutonPoints`)
9. **`AutonTotalShootingTime`** (Double, 2 decimals) - Total shooting time during Auton.
10. **`AutonAmountOfShooting`** (Integer) - Total number of shooting instances/cycles.
11. **`AutonClimb`** (Integer: 1 or 0) - `1` if the robot climbed during Auton, `0` otherwise.
12. **`AutonPassing`** (Integer) - Number of passing actions during Auton.

### 3. TeleOp Data (`TeleOpPoints`)
13. **`TeleOpTotalShootingTime`** (Double, 2 decimals) - Total shooting time during TeleOp.
14. **`TeleOpTotalAmount`** (Integer) - Total shooting instances/cycles during TeleOp.
15. **`TeleOpDefense`** (Integer: 1 or 0) - `1` if the robot played defense, `0` otherwise.
16. **`TeleOpNeutralTrips`** (Integer) - Number of trips to the neutral zone.
17. **`TeleOpPushBalls`** (Integer) - Number of balls pushed.
18. **`TeleOpPassing`** (Integer) - Number of passing actions during TeleOp.

### 4. Endgame Data (`EndPoints`)
19. **`EndClimbStatus`** (Integer) - Final climb state (0 = None, 1-9 = Level IDs).
20. **`EndPark`** (Integer: 1 or 0) - `1` if parked, `0` otherwise.
21. **`EndPushBalls`** (Integer) - Number of balls pushed during endgame.
22. **`EndPassing`** (Integer) - Number of passing actions during endgame.
23. **`EndRobotBroken`** (Boolean) - Whether the robot broke during the match.
24. **`EndNeutralTrips`** (Integer) - Number of neutral zone trips during endgame.
25. **`EndShootingAccuracy`** (Integer) - Shooting accuracy metric (0-100).
26. **`EndEndgameTime`** (Double) - Time spent performing endgame actions.
27. **`EndShootingCycles`** (Integer) - Number of shooting cycles during endgame.
28. **`EndComments`** (String, quoted) - Free-text comments. Wrapped in double quotes; internal quotes are escaped as `""`.

**Total: 28 columns.**

*Note: The CSV values are comma-separated with no spaces between fields. The Comments field is always quoted to safely handle commas in free text.*
